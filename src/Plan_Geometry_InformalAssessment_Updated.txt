## **4. Drawing Task Approach (Avoiding Precision Measurement Issues)**

### **Problem with Traditional Drawing Tasks:**
- Precision measurement is impossible on various screen sizes
- Even "parallel" lines require tolerance levels
- Students can't accurately draw specific angles or lengths
- Evaluation becomes subjective and frustrating

### **Alternative Task Types (No Precision Required):**

#### **A. Binary Choice Tasks Instead of Drawing**
- "Click on the line that is parallel to this one"
- "Select which line is perpendicular"
- "Choose the correct completion of this triangle"
- "Click on all the right angles in this shape"

#### **B. Template-Based Drawing**
- Provide pre-drawn elements that students can drag/place
- "Drag this line to make it parallel"
- "Place this angle to complete the triangle"
- "Select the correct side length from these options"
- "Drag the hash marks to the equal sides"

#### **C. Visual Recognition Tasks**
- "Click on all the right angles in this shape"
- "Select which sides are equal"
- "Identify the parallel sides"
- "Circle the acute angles"

#### **D. Construction with Guides**
- Show grid lines or guides
- Provide snap-to-grid functionality
- Use visual indicators (like in CAD programs)
- "Complete this shape using the grid"

#### **E. Marking/Annotation Tasks (Simple)**
- "Mark the equal sides with hash marks" (drag pre-made marks)
- "Circle the right angle"
- "Label the vertices A, B, C" (drag labels)

### **Examples of Practical Tasks:**
- "Click on the line that is parallel to AB"
- "Drag the hash marks to show equal sides"
- "Select which angle is obtuse"
- "Place the correct label on each vertex"
- "Choose the line that completes this triangle"

## **5. Response Evaluation Strategy**

### **MCQ Questions**
- **Method**: Simple string matching
- **AI Needed**: No
- **Implementation**: Cached correct answers

### **Drag & Drop Questions**
- **Method**: Position/placement matching
- **AI Needed**: No
- **Implementation**: Cached correct positions

### **Alternative Task Evaluation (No Precision Drawing)**

#### **Binary Choice Tasks**
- **Method**: Click/selection validation
- **AI Needed**: No
- **Implementation**: Cached correct selections

#### **Template-Based Tasks**
- **Method**: Drag-and-drop position validation
- **AI Needed**: No
- **Implementation**: Cached correct positions

#### **Visual Recognition Tasks**
- **Method**: Element selection validation
- **AI Needed**: No
- **Implementation**: Cached correct elements

#### **Grid-Based Construction**
- **Method**: Grid position validation
- **AI Needed**: No
- **Implementation**: Grid coordinate matching

### **Evaluation Examples:**
```javascript
// Binary choice evaluation
const evaluateBinaryChoice = (task, userSelection) => {
  return userSelection === task.correctElement;
};

// Template-based evaluation
const evaluateTemplate = (task, userPlacement) => {
  return isCorrectPlacement(userPlacement, task.correctPosition);
};

// Visual recognition evaluation
const evaluateRecognition = (task, userSelections) => {
  return arraysEqual(userSelections, task.correctElements);
};
```

## **7. User Experience Flow**
1. **Mode Selection**: Choose "Explore" or "Informal Assessment"
2. **Question Presentation**: Shape + question + interaction area
3. **Hint System**: Optional hints before answering (user choice)
4. **Answer Submission**: MCQ click, binary choice selection, visual recognition, or template-based completion
5. **Feedback**: Immediate right/wrong + explanation (always shown)
6. **Progress**: Track completed questions (20 total)

## **8. Technical Implementation Approach**

### **Architecture Overview:**
- **Enhance Current System**: Add question mode to existing Geometry Studio
- **Reuse Backend**: Same Python rendering with enhanced parameters (3x size)
- **New Components**: Question wrapper, hint system, progress tracking, task-specific UI components
- **Firebase Storage**: 20 questions per grade per visual aid
- **Evaluation**: Start with rule-based, add AI later for complex tasks

### **Component Architecture:**

### **Data Flow:**
```javascript
1. User selects "Informal Assessment" mode
2. System loads question from Firebase: GET /api/questions/grade/7/topic/angles
3. Backend generates geometry diagram: GET /api/geometry/generate-diagram
4. Frontend renders QuestionWrapper with appropriate TaskInterface
5. User interacts with task (click, drag, select)
6. Frontend validates locally, shows immediate feedback
7. System submits answer: POST /api/questions/submit-answer
8. Backend evaluates: POST /api/geometry/evaluate-task
9. Frontend shows explanation and loads next question
```
