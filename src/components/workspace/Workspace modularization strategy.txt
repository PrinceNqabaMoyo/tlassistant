# 1) Introduce a “Registry” (one-time plumbing)

## 1.1 Create a registry file
Create:

- `src/components/workspace/workspaceRegistry.js`

It exports a map:

- key: `workspaceMode` (unchanged strings like `grade7_exponents_scaffold`)
- value: `{ render, getVisualAids? }`

Example shape (conceptually):

- **`render(ctx)`** returns the correct component for that mode
- `ctx` includes the shared props already used across topics (like `renderMathText`, navigation helpers, etc.)

## 1.2 Workspace.jsx becomes a shell
In [Workspace.jsx](cci:7://file:///c:/Users/princ/fundile-tlassistant-vite/src/components/workspace/Workspace.jsx:0:0-0:0) we keep only:
- global UI + layout
- `workspaceMode` switching
- shared helpers (like `renderMathText`, any shared modals if they truly are shared)
- a single selection point:

```
const entry = workspaceRegistry[workspaceMode];
return entry ? entry.render(ctx) : <DefaultWorkspace />;
```

**Rule**: [Workspace.jsx](cci:7://file:///c:/Users/princ/fundile-tlassistant-vite/src/components/workspace/Workspace.jsx:0:0-0:0) should not grow again. New topics go into the registry + a topic folder.

# 2) “Controller module” pattern (applies to every topic)

For each topic, create a **controller module** that owns all the state/fetching/effects previously living in [Workspace.jsx](cci:7://file:///c:/Users/princ/fundile-tlassistant-vite/src/components/workspace/Workspace.jsx:0:0-0:0).

## 2.1 Create `controller.js`
Path example:

- `src/components/workspace/grade7/mathematics/exponents/controller.js`

It exports:

- `useGrade7MathExponentsController(params)`

Where `params` includes only the minimum:
- `workspaceMode`
- `renderMathText` (if needed)
- any shared API base helpers already used
- optional: `seed` policy if you have one

The controller returns an object:

- **State**
  - `difficulty`
  - `loading`
  - `error`
  - `questions` (practice set) / `question` (scaffold single)
  - `answers`, `feedback`
  - `stepIndex` for scaffold
  - `visualAidsOpen`, `activeVisualAidTab` (if used)

- **Actions**
  - `setDifficulty`
  - `fetchScaffoldQuestion({ subskill, difficulty })`
  - `fetchPracticeSet({ subskill, difficulty, count })`
  - [setAnswer(qIndex, value)](cci:1://file:///c:/Users/princ/fundile-tlassistant-vite/src/components/workspace/grade10/Grade10PatternsSequencesPractice.jsx:66:4-74:6) / [checkAnswer()](cci:1://file:///c:/Users/princ/fundile-tlassistant-vite/src/components/workspace/grade11/Grade11PatternsSequencesScaffold.jsx:81:4-91:6) (if this logic lives in Workspace; if it lives in the component already, keep it there)

- **Static config**
  - `scaffoldSteps` array
  - `visualAids` (optional)

## 2.2 Move `useEffect` triggers into the controller
Any logic like:
- “when workspaceMode becomes `..._scaffold`, fetch the current step”
- “when stepIndex changes, refetch”
moves into the controller.

**Rule**: [Workspace.jsx](cci:7://file:///c:/Users/princ/fundile-tlassistant-vite/src/components/workspace/Workspace.jsx:0:0-0:0) should not have per-topic `useEffect` anymore after migration.
